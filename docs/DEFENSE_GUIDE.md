# Подготовка к защите проекта FA Schedule Windows

Этот документ написан максимально простым языком. Его можно открыть перед защитой и пройтись по нему как по шпаргалке: что показать, что сказать, какие файлы открыть и как отвечать на типичные вопросы.

Проект — Windows-приложение на Flutter для просмотра расписания Финансового университета через RUZ API. Пользователь ищет группу или преподавателя, выбирает результат, а приложение загружает расписание и показывает его в недельном календаре.

## 1. Очень коротко: что делает приложение

Приложение помогает посмотреть расписание занятий. Оно не хранит расписание внутри себя, а получает актуальные данные с сервера `ruz.fa.ru`.

В интерфейсе можно:

- искать учебные группы;
- искать преподавателей;
- открывать расписание выбранной группы;
- открывать расписание выбранного преподавателя;
- переключать недели;
- смотреть подробности каждой пары;
- при необходимости включать локальный прокси `localhost:3000`.

Главный сценарий пользователя: запустить приложение, ввести название группы или фамилию преподавателя, выбрать найденный вариант и увидеть расписание на неделю.

## 2. Что показать на защите по шагам

1. Запустить приложение:

```powershell
run -d windows
```

2. Показать главный экран поиска.
3. Выбрать режим поиска `Группы`.
4. Ввести пример группы, например `ПИ21`.
5. Выбрать найденную группу из списка.
6. Показать экран расписания и недельный календарь.
7. Нажать стрелку вперед и показать следующую неделю.
8. Нажать стрелку назад и показать предыдущую неделю.
9. Нажать кнопку `Сегодня`, чтобы вернуться к текущей неделе.
10. Показать режим поиска `Преподаватели`.
11. Ввести фамилию преподавателя, например `Андропов`.
12. Открыть расписание преподавателя.
13. Открыть настройки API и показать переключатель `localhost:3000`.
14. Объяснить, что приложение умеет работать напрямую с `ruz.fa.ru` и через локальный прокси.

## 4. Архитектура простыми словами

Код разделен на слои. Это делается для порядка: каждый слой отвечает за свою часть работы, поэтому проект легче понимать, изменять и объяснять на защите.

- `ui` — показывает экраны, кнопки, поля ввода, календарь и карточки занятий.
- `state` — хранит состояние приложения и управляет действиями пользователя.
- `services` — отправляет HTTP-запросы к RUZ API.
- `models` — превращает JSON из API в понятные Dart-объекты.

Главная мысль: **интерфейс не делает запросы к серверу напрямую**.

```text
Пользователь
  -> UI
  -> ScheduleController
  -> RuzApiClient
  -> RUZ API
  -> модели ScheduleTarget / ScheduleLesson
  -> ScheduleController
  -> UI
```

## 5. Как проходит поиск

1. Пользователь печатает текст в `TargetSearchPanel`.
2. Виджет ждет `450` миллисекунд debounce-паузы.
3. `TargetSearchPanel` вызывает `ScheduleController.search()`.
4. `ScheduleController` ставит статус `loading`.
5. `ScheduleController` вызывает `RuzApiClient.searchTargets()`.
6. `RuzApiClient` отправляет `GET /api/search`.
7. Ответ JSON превращается в список `ScheduleTarget`.
8. `ScheduleController` сохраняет `searchResults`.
9. `Provider` уведомляет Flutter, и список на экране обновляется.

## 6. Как открывается расписание

1. Пользователь нажимает на результат поиска.
2. `HomeScreen` открывает маршрут `/schedule/:kind/:id`.
3. `go_router` создает `ScheduleScreen`.
4. `ScheduleScreen` просит `ScheduleController` загрузить расписание.
5. Контроллер вызывает `RuzApiClient.loadSchedule()`.
6. API возвращает JSON со списком занятий.
7. Каждый элемент JSON превращается в `ScheduleLesson`.
8. `ScheduleWeekGrid` раскладывает занятия по дням недели.

## 7. Основные файлы проекта

### `lib/main.dart`

Точка входа. Здесь создается `RuzApiClient`, подключается `Provider`, создается `ScheduleController`, настраивается тема и маршруты `go_router`.

### `lib/state/schedule_controller.dart`

Главный контроллер состояния. Он хранит тип поиска, результаты поиска, выбранную неделю, выбранную группу или преподавателя, список занятий, статусы загрузки, ошибки и флаг прокси.

### `lib/services/ruz_api_client.dart`

Сетевой слой. Здесь собраны базовый адрес, адрес локального прокси, endpoint поиска, endpoint расписания, query-параметры, проверка HTTP-ответа и преобразование JSON.

### `lib/models/schedule_target.dart`

Модель результата поиска. Один объект `ScheduleTarget` означает: пользователь может выбрать эту группу или этого преподавателя.

Поля:

- `id` — идентификатор в RUZ API;
- `title` — название группы или имя преподавателя;
- `subtitle` — дополнительное описание;
- `kind` — группа или преподаватель.

### `lib/models/schedule_lesson.dart`

Модель одного занятия. Она забирает из большого JSON только то, что нужно показать в интерфейсе: дату, время, дисциплину, тип занятия, аудиторию, корпус, преподавателя, группу и примечание.

### `lib/ui/screens/home_screen.dart`

Главный экран поиска. Он показывает `TargetSearchPanel` и открывает расписание после выбора результата.

### `lib/ui/screens/schedule_screen.dart`

Экран расписания. Здесь находится верхняя панель с названием выбранного объекта, кнопки переключения недель, кнопка `Сегодня`, кнопка обновления, календарь и список занятий на широком экране.

### `lib/ui/widgets/schedule_week_grid.dart`

Недельная сетка. Она строит семь дней недели и размещает карточки занятий по датам.

### `tool/ruz_proxy_server.dart`

Локальный прокси-сервер. Он нужен как запасной вариант, если прямой доступ к `ruz.fa.ru` нестабилен.

## 8. API структура подробно

Приложение использует два типа запросов: поиск и загрузку расписания.

### Поиск группы

```http
GET /api/search?term=<строка>&type=group
```

Пример:

```text
https://ruz.fa.ru/api/search?term=ПИ21&type=group
```

### Поиск преподавателя

```http
GET /api/search?term=<строка>&type=person
```

Пример:

```text
https://ruz.fa.ru/api/search?term=Андропов&type=person
```

### Расписание группы

```http
GET /api/schedule/group/<id>?start=yyyy.MM.dd&finish=yyyy.MM.dd
```

### Расписание преподавателя

```http
GET /api/schedule/person/<id>?start=yyyy.MM.dd&finish=yyyy.MM.dd
```

`id` берется из результата поиска. `start` и `finish` формируются в `DateRange`. Формат даты:

```text
yyyy.MM.dd
```

Например:

```text
2026.05.31
```

## 9. Почему JSON превращается в модели

Сырой JSON неудобно использовать прямо в интерфейсе:

- поля могут быть `null`;
- некоторые поля могут отсутствовать;
- у группы и преподавателя немного разная структура;
- в JSON много лишних данных;
- UI становится сложнее читать.

Поэтому ответ API сначала превращается в модели:

- `ScheduleTarget` — результат поиска;
- `ScheduleLesson` — одно занятие.

## 10. Provider в проекте

`Provider` нужен, чтобы разные экраны могли читать одно и то же состояние.

Например:

- экран поиска читает `searchResults`;
- экран расписания читает `lessons`;
- настройки API читают `proxyEnabled`;
- шапка расписания читает количество пар и диапазон недели.

В `main.dart` подключен:

```dart
ChangeNotifierProvider(
  create: (_) => ScheduleController(apiClient: apiClient),
  child: const FaScheduleApp(),
)
```

`context.watch` используется, когда виджет должен перерисоваться при изменении состояния.

`context.read` используется, когда нужно просто вызвать действие:

```dart
context.read<ScheduleController>().search(value);
```

## 11. go_router в проекте

`go_router` отвечает за переходы между экранами.

В проекте два главных маршрута:

```text
/                    -> HomeScreen
/schedule/:kind/:id  -> ScheduleScreen
```

`kind` показывает тип объекта:

- `group` — группа;
- `person` — преподаватель.

`id` — идентификатор группы или преподавателя в RUZ API.

## 12. Debounce в поиске

Debounce — задержка перед поиском. Пока пользователь быстро печатает, приложение не отправляет запрос на каждую букву. Запрос отправляется только после короткой паузы.

Зачем это нужно:

- меньше запросов к API;
- интерфейс не дергается;
- результаты поиска обновляются спокойнее;
- приложение выглядит аккуратнее.

Файл:

```text
lib/ui/widgets/target_search_panel.dart
```

## 13. Прокси

Прокси — маленький локальный сервер. Приложение может отправить запрос на `localhost:3000`, а прокси пересылает его на `https://ruz.fa.ru` и возвращает ответ обратно.

Запуск:

```powershell
dart run tool/ruz_proxy_server.dart
```

После запуска прокси в приложении нужно открыть настройки API и включить переключатель `localhost:3000`.

## 14. Частые вопросы и готовые ответы

### Где происходит запрос к серверу?

В файле `lib/services/ruz_api_client.dart`, в методах `searchTargets()` и `loadSchedule()`.

### Где хранится состояние?

В файле `lib/state/schedule_controller.dart`, в классе `ScheduleController`.

### Где JSON превращается в объекты?

В файлах `lib/models/schedule_target.dart` и `lib/models/schedule_lesson.dart`.

### Что будет, если API не ответит?

`RuzApiClient` выбросит `RuzApiException`, `ScheduleController` поймает ошибку, сохранит текст в `errorMessage`, а интерфейс покажет сообщение пользователю.

### Почему приложение считается Windows-приложением?

Потому что оно запускается на Windows device через Flutter desktop, собирается Visual Studio C++ toolchain и имеет Windows runner.

### Что делать, если расписание пустое?

Это не обязательно ошибка. У выбранной группы или преподавателя может не быть занятий на текущей неделе. Нужно переключить неделю назад или вперед.

### Почему не использован готовый проект из GitHub?

Потому что задание требовало самостоятельную реализацию. Референс использовался только для понимания API и этапов, а дизайн, структура и код приложения сделаны отдельно.

## 17. Если на защите что-то пошло не так

### Не запускается приложение

Проверить:

```powershell
flutter doctor
```

Для Windows должны быть готовы Flutter и Visual Studio.

### Не грузится расписание

Проверить интернет и доступность `ruz.fa.ru`.

Если прямой API не работает, запустить прокси:

```powershell
dart run tool/ruz_proxy_server.dart
```

Потом включить `localhost:3000` в настройках приложения.

### Расписание пустое

Переключить неделю вперед или назад. У выбранной группы или преподавателя может не быть занятий на текущей неделе.
